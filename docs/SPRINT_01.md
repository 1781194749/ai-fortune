# 玄机 AI Sprint 01 开工计划

## 当前目标

把“能收费、能体验、能沉淀报告”的 MVP 主链路跑通，并让 AI 工具、图片上传、支付适配和后台观测形成可演示闭环。

## 已落地

- 品牌：首页、黑金视觉、首屏图、响应式布局
- 账号：邮箱验证码登录、签名 cookie session
- 收费：会员套餐、星力余额、mock payment 订单、支付成功发放
- 会员档案：基础信息保存、八字/五行摘要派生、AI 对话读取档案
- 工具：塔罗推演、八字五行简析、推演步骤展示
- 报告：报告保存、会员中心报告列表、报告详情页、报告状态、分享 slug、订单绑定
- 持久化：订单、钱包、会员状态、报告优先写入 Prisma/PostgreSQL，本地无数据库时回退内存
- AI 编排：OpenAI Responses API 封装、工具调用轨迹、UsageLog 成本记录、本地降级回答
- 手相上传：七牛上传凭证、前端直传、图片 metadata、OpenAI 视觉模型入口、本地降级分析、删除档案
- 支付适配：支付宝/微信支付预下单、异步通知、验签入口和真实支付参数开关
- 优惠转化：支持 `FIRST50`、`XUANJI20`、`SHARE15` 优惠码，订单按折扣实付价创建并记录促销事件；优惠码具备有效期、总量上限、单用户限用和未支付订单 30 分钟额度保留规则
- 优惠配置：后台可启用/暂停优惠码，调整总量、单用户限制和有效期，并可恢复默认配置
- 首单实验：新用户按 userId 确定性分流到 `FIRST50` 或 `XUANJI20`，会员购买页自动预填优惠码，并记录曝光、下单、支付和实收；后台可根据建议固化胜出变体为默认新客券，也可恢复 A/B
- 八卦问事：起卦算法、本卦/变卦、动爻解释、星力扣费和报告沉淀
- 报告分享：报告详情页可开启/关闭公开分享，关闭后旧公开链接失效
- 深度报告：支付后先创建生成中报告，后台生成正文，前端轮询更新到完成或失败
- 报告导出：已完成报告提供私有导出版，可通过浏览器打印保存 PDF
- 公开传播：分享页支持复制链接、系统分享、生成公开报告海报、二维码和回流购买入口
- 分享追踪：记录分享页访问、海报访问、二维码来源、复制/下载等传播事件
- 分享归因：公开报告落地后写入归因 cookie，并记录登录、下单和 mock 支付转化
- 增长 ROI：后台按分享来源聚合落地、登录、下单、支付、实收、让利和收入/让利倍数
- 渠道策略：后台按自然传播、海报回流、私域社群、达人/投放等分层来源，并给出预算、优惠和内容动作
- 渠道复盘：后台支持导出渠道投放 CSV，包含渠道汇总、来源明细、支付转化、实收、让利和收入/让利倍数
- 渠道治理：分享落地支持 `source` 与 `utm_source/utm_medium/utm_campaign`，后台提示未知来源、需归一来源和推荐命名格式
- 投放成本：后台可按 source 生成投放链接模板并录入预算/成本，ROI 报表和 CSV 同步展示投放成本、净回收和收入/投放倍数
- 预算预警：渠道预算支持周期开始/结束时间，并根据周期、支付订单、投放成本和综合回收倍数生成高风险/关注/观察预警
- 预警配置：后台可调整收支平衡倍数、健康回收倍数、临期天数、无支付落地阈值和高成本阈值
- 复盘归档：后台可把渠道预算周期归档为加码、暂停、复测或结案，并将最近复盘结论写入渠道 CSV
- 复盘筛选：后台可按加码、暂停、复测、结案筛选复盘档案，并导出当前筛选 CSV
- 增长洞察：后台根据来源漏斗、ROI、优惠码额度和待支付占用给出加码、优化、收券或观察建议
- 生产化观测：后台显示优惠码配置、首单实验策略和后台审计当前处于数据库持久化、数据库回退内存或纯内存模式
- 落库验收：后台健康页可运行 PostgreSQL 读写探针，区分未配置数据库、数据库不可用降级和真实 UsageLog 写入读回成功，并展示上线关键事件的持久化覆盖
- 数据库落地计划：后台健康页和 `/api/admin/launch/database-plan` 输出 PostgreSQL 实例、DATABASE_URL、数据库连接检查、Prisma Schema、落库探针、上线事件覆盖、备份回滚、命令顺序和证据清单，并支持快填连接、迁移、探针、备份和恢复演练证据；`launch:db-check -- --base-url=...` 可运行时确认数据库计划 API 步骤和命令组完整
- 域名与部署落地计划：后台健康页和 `/api/admin/launch/deployment-plan` 输出域名/DNS、HTTPS APP_URL、公网 URL 验收、部署平台变量、后台访问保护、会话密钥、公网回调、预检脚本、页面烟测、重启回滚和证据清单，并支持快填正式域名、生产变量、后台保护、公网回调、预检、页面烟测和回滚证据；`launch:url-check -- --base-url=...` 可运行时确认域名/部署计划 API 步骤和命令组完整
- 第三方诊断：后台健康页可运行 OpenAI 模型读取、七牛上传 token/上传域名、支付宝签名、微信支付签名探针
- AI 与图片落地计划：后台健康页和 `/api/admin/launch/ai-storage-plan` 输出 OpenAI 项目、模型变量、成本费率、`launch:ai-storage-check` 命令行检查、模型诊断、七牛 bucket、七牛变量、CORS、手相视觉、深度报告和 AI 成本样本步骤，并支持 AI/图片验收证据快填；`launch:ai-storage-check -- --base-url=...` 可运行时确认 AI/图片计划 API 步骤和命令组完整
- 上线总闸：后台健康页聚合生产清单、外部事项、落库验收和第三方诊断，输出 Go / No-Go 状态与优先处理项
- 上线 Runbook：后台健康页按基础安全、生产数据、AI/图片、支付、账号和合规输出执行清单、负责人、动作和验收证据
- 上线预检：`.env.production.example` 提供生产变量模板，`npm run launch:secrets` 生成基础安全密钥，`npm run launch:preflight` 在部署前检查域名、密钥、数据库、支付和合规阻断项
- 上线证据：后台健康页可把当前 Go/No-Go、Runbook、落库探针、数据库验收证据、部署验收证据、第三方诊断、AI/图片验收证据、端到端验收留证、真实支付小额验收、AI 成本样本、今日动作执行记录和目标推进摘要归档到 UsageLog，作为上线前复核记录
- 外部事项：后台健康页可跟踪主体、域名、ICP备案、生产数据库、OpenAI、七牛、微信开放平台、微信支付、支付宝和法务审查状态、目标日期、提交回执、证据链接和证据备注
- 收费上线包：后台健康页聚合 Go/No-Go、Runbook、外部事项、端到端验收留证、数据库验收证据、部署验收证据、AI/图片验收证据、真实支付验收、AI 成本样本、目标推进记录、今日动作执行记录和最新证据归档，输出上线包状态、必补事项、证据缺口和下一步动作，并在 Go/No-Go、端到端验收、数据库验收、部署验收、AI/图片验收、支付验收、成本样本、目标推进记录或今日动作执行记录变化后提示刷新归档
- 办理资料包：后台健康页把外部事项拆成准备材料、办理步骤、产出凭证、生产变量和验收方式，方便你同步推进主体、备案、云服务和支付申请
- 创始人办理包：后台健康页提供主体路径建议、主体路径决策助手、关键办理顺序、材料产物、解锁变量和官方入口，用于线下推进主体、备案、支付和云服务
- 线下办理行动包：后台健康页和 `/api/admin/launch/offline-action-pack` 聚合创始人办理包、平台申请材料、生产变量批次和排期风险，输出今天先办、阶段状态、材料、填表字段、生产变量、回执证据、建议目标日和可复制办理一页纸，并在顶部快填当前事项或服务端批量保存优先动作的状态、目标日期、回执、证据链接和备注；`launch:offline-action-check -- --base-url=...` 可运行时确认线下办理行动包、主体路径决策、外部事项和后台区块完整
- 生产变量核对：后台健康页按缺失、占位、本地值、强度不足和开关不符核对生产变量，密钥和连接串只展示脱敏状态
- 变量草案：后台健康页可生成安全的 `.env.production` 草案、优先填写变量和平台来源提示，密钥与连接串只输出占位符
- 生产变量批次清单：后台健康页和 `/api/admin/launch/env-batch-plan` 按基础安全、主体域名、生产数据库、OpenAI、七牛、真实支付和微信登录输出填写顺序、验证动作和证据口径，基础安全批次包含密钥生成命令，真实支付批次包含内部小额订单白名单
- 变量闸门：生产变量核对结果已并入 Go/No-Go 和 Runbook，变量 blocking 会阻断上线包并进入证据归档
- 证据增强：上线证据归档会保存生产上线总门禁、生产变量核对摘要、分组状态、优先变量缺口、数据库验收证据、部署验收证据、AI/图片验收证据、端到端验收留证、支付验收状态、最近支付证据、AI 成本样本、今日动作执行记录、线下办理当前动作、30/60/90 目标推进记录和阶段推进门槛 `transitionGate.canAdvance`，方便上线前复盘
- 上线交接：后台健康页提供一份可复制交接摘要，把收费上线包、生产上线总门禁、目标后续推进复盘、阶段推进门槛、`transitionGate.canAdvance`、线下办理当前动作、目标补齐入口、生产变量、外部办理和证据归档压缩成当前目标、总门禁结论、阶段放行判断、线下办理今日先办事项、目标推进缺口、阻断项和下一步动作
- 执行工作计划：后台健康页按外部办理、平台申请、生产变量、联调验收、单位经济和证据放量六条工作线展示负责人、依赖、动作和验收证据
- 本周推进看板：后台健康页把执行工作计划、排期风险、创始人办理包、平台申请材料和灰度阶段合并成本周目标、负责人、截止日和验收证据，并同时展示本周阶段门槛 `transitionGate.canAdvance`，支持建议承诺日、承诺状态、批量保存任务承诺与查看承诺覆盖率
- 本周推进验收：`npm run launch:weekly-focus-check` 静态复核本周推进看板、承诺覆盖率、负责人分组、本周承诺持久化、GET/PATCH API、后台承诺表单和文档口径；带 `-- --base-url=...` 时复核 `/api/admin/launch/weekly-focus`、本周承诺 PATCH 写回和后台本周推进区块
- 灰度放量计划：后台健康页按资质准备、生产配置、联调验收、小额订单和放量复盘五阶段展示进入条件、退出证据、平台申请阻断和单位经济风险
- 排期风险：后台健康页根据外部事项目标日期提示逾期、临期和未排期事项，并给未排期事项建议目标日
- 回调配置：后台健康页根据 APP_URL 生成支付宝、微信支付、七牛 CORS、协议材料和微信开放平台配置清单
- 平台申请材料：后台健康页按 ICP、支付宝、微信支付、微信开放平台、七牛和 OpenAI 输出填表字段、官方入口、生产变量、提交回执、证据链接和证据要求
- 单位经济：后台健康页检查产品价格、星力发放、单次付费功能和 AI 成本日志，提示毛利复盘缺口，并支持快填功能、模型、tokens、成本金额和证据链接作为成本样本
- 商业模型：后台健康页和 `/api/admin/launch/business-model` 聚合用户画像、主推商品、AI 成本占比、贡献毛利、建议获客成本上限和渠道回收护栏，用于判断谁会买单、先推哪个商品、投放预算是否可控
- 合规核对：后台健康页检查主体、备案、协议入口、免责声明、隐私供应商披露、图片授权和法务复核状态；`launch:compliance-check` 可命令行复核协议四件套、主体备案变量、合规 API 和后台计划 wiring
- 合规与主体落地计划：后台健康页和 `/api/admin/launch/compliance-plan` 按主体路径、域名/ICP备案、协议主体、支付主体一致、隐私供应商、图片授权、退款口径和法务归档输出步骤、办理顺序和证据清单；`launch:compliance-check -- --base-url=...` 可运行时确认协议页、合规核对 API 和计划命令组完整
- 验收矩阵：后台健康页按登录、会员档案、AI 对话、命理工具、手相、支付、深度报告、分享归因和后台证据列出端到端手测用例，并支持保存验收结果、验收人、截图/录屏链接和备注
- 证据行动中心：后台健康页和 `/api/admin/launch/evidence-action-center` 按截图/录屏、平台回执、小额订单、成本样本、后台归档和后台记录输出补证负责人、目标、优先项和可复制行动口径
- 证据缺口：后台健康页聚合上线包、验收矩阵、平台申请材料、单位经济、真实支付和最新归档，显示验收可执行率、证据归档状态、补证类型分类和优先补证清单
- 证据链验收：`npm run launch:evidence-check` 静态复核上线证据行动中心、上线证据缺口、上线证据归档 metadata、API 鉴权/no-store、后台归档按钮和文档口径；带 `-- --base-url=...` 时复核 `/api/admin/launch/evidence-action-center`、`/api/admin/launch/evidence-gap`、`/api/admin/launch/evidence`、归档 POST 和后台证据区块
- 支付小额验收：后台健康页聚合支付宝/微信支付配置、真实订单、PAID 回调、平台交易号、钱包流水和验收证据记录，输出支付验收缺口、证据快填和可复制核对清单
- 支付落地计划：后台健康页和 `/api/admin/launch/payment-plan` 按支付宝/微信支付输出资质申请、商户参数、`launch:payment-check` 命令行检查、签名诊断、回调业务字段校验、微信支付 API v3 resource 解密烟测、回调地址、小额订单、PAID 回调、权益到账、对账证据和配置顺序；`launch:payment-check -- --base-url=...` 可运行时确认每个渠道都有 `callback_guard` 步骤
- 资质接入验收：`npm run launch:qualification-check` 静态复核 ICP/支付资质接入前置验收、外部事项 GET/PATCH、平台申请材料、合规主体计划、支付落地计划、真实支付入口保护和文档口径；带 `-- --base-url=...` 时复核 `/api/admin/launch/external-readiness`、`/api/admin/launch/application-pack`、`/api/admin/launch/compliance-plan`、`/api/admin/launch/payment-plan`、`/api/admin/launch/decision` 和后台资质区块

- 日报验收脚本：`npm run launch:daily-brief-check` 静态复核今日目标推进日报、今日优先动作来源、今日动作执行持久化、日报 GET/PATCH API、后台快填和文档口径；带 `-- --base-url=...` 时复核 `/api/admin/launch/daily-brief`、今日动作 PATCH 写回和后台日报区块

- 目标规划验收：`npm run launch:goal-plan-check` 静态复核 30/60/90 目标规划、四阶段目标、阶段推进门槛、目标推进持久化、GET/PATCH API、后台目标推进快填和文档口径；带 `-- --base-url=...` 时复核 `/api/admin/launch/goal-plan`、当前阶段 PATCH 写回和后台目标规划区块
- 交接验收：`npm run launch:handoff-check` 静态复核上线交接与执行计划验收、上线交接摘要、六条执行工作线、交接/执行计划 API、后台交接区块和文档口径；带 `-- --base-url=...` 时复核 `/api/admin/launch/handoff`、`/api/admin/launch/workplan`、生产总门禁交接、线下办理交接、目标后续推进和可复制交接口径
- 真实支付保护：会员页和 `/api/payments/live/orders` 会根据最终上线决策关闭或放行真实预下单，`paid_smoke` 阶段只允许内部白名单账号小额下单，避免 no_go 或内部灰度状态误收款

- 上线自检：生产环境变量、支付、七牛、OpenAI、后台保护和备案主体检查
- 文档：PRD、设计规范、技术架构、任务清单、执行路线

## 下一轮优先级

1. 生产数据库：先看 `/admin/health` 的数据库落地计划，配置 Postgres 后运行 `npm run launch:db-check` 验证连接，再执行 `npm run prisma:migrate:deploy`，并用 `npm run launch:db-check -- --schema` 验核心表，最后运行落库探针；通过数据库验收证据快填保存连接、迁移、探针、备份和恢复演练证据，确认外部事项、本周承诺、今日动作执行记录、目标推进记录、第三方诊断、AI/图片验收、AI 成本样本、数据库验收和上线证据都有持久化记录。
2. 域名与部署：先看 `/admin/health` 的生产变量批次清单和域名与部署落地计划，运行 `npm run launch:secrets` 生成后台 token 与会话密钥，完成域名/DNS、HTTPS APP_URL、部署平台变量、公网回调、页面烟测和回滚证据，再运行 `npm run launch:url-check` 与 `npm run launch:preflight` 直到无 blocking；通过部署验收证据快填保存 URL 检查、预检输出、页面烟测和回滚记录。
3. 外部事项：在 `/admin/health` 先看线下办理行动包确认今天先办事项，运行 `npm run launch:offline-action-check` 复核材料模板、主体路径、外部事项和后台区块，再看创始人办理包里的主体路径决策助手，确认第一版国内收费主体路径，随后跟踪主体、域名、ICP备案、支付商户、七牛和 OpenAI 办理状态、目标日期、提交回执、证据链接和证据备注，并按办理资料包准备材料、变量和验收凭证；处理中和已提交只表示进度，未完成前仍进入 Go/No-Go 阻断。
4. 上线 Runbook：逐项收集 `/admin/health` 中的负责人动作和验收证据，直到无 blocking。
5. 上线交接复核：在 `/admin/health` 先看上线阻断总控台和今日目标推进日报确认今天先办动作，并用今日动作执行快填记录状态、负责人、证据备注和推进备注；再看目标后续推进复盘判断下一步先补执行记录、本周承诺、证据归档还是阶段衔接，并通过补齐入口跳到对应填写区；随后复核 30/60/90 天目标规划并保存阶段目标日、负责人、推进状态和证据备注；再复核上线交接摘要、上线执行工作计划、分阶段放量计划、排期风险、域名与部署落地计划、回调配置清单、合规核对、合规与主体落地计划、端到端验收矩阵、真实支付小额订单验收、证据行动中心和证据缺口，按收费上线包必补事项处理阻断项，并在完成生产变量、落库、第三方诊断、支付小额验收和主链路手测后归档当前证据。
6. 增长增强：真实转化数据接入后的复盘样本沉淀和投放结论校准。
7. 真实图片 AI 联调：先看 `/admin/health` 的真实联调排程和 AI 与图片能力落地计划，配置 OpenAI Key、`OPENAI_VISION_MODEL`、七牛公开域名和 OpenAI 每百万 token 成本变量后运行 `npm run launch:ai-storage-check` 和后台诊断，并通过 AI/图片验收证据快填保存诊断、手相视觉、深度报告和成本样本证据。
8. 真实支付联调：主体资质完成后先看 `/admin/health` 的真实联调排程和支付落地计划，填入支付宝/微信支付商户参数和 `LIVE_PAYMENT_SMOKE_TEST_USER_IDS` 或 `LIVE_PAYMENT_SMOKE_TEST_EMAILS`，运行 `npm run launch:payment-check` 和后台诊断，再用内部账号做沙箱/小额订单联调，至少先闭合一个渠道。
9. 法务审查与真实资质：先看 `/admin/health` 的合规与主体落地计划，运行 `npm run launch:compliance-check` 核对协议、主体、备案和计划 wiring，再按主体路径、域名/ICP备案、支付主体一致、退款客服口径和法务归档逐项办理。
